=== What the new tests cover ===

TestResponseStructure (4 tests)
  - Every result (SAFE, WARNING, HIGH_RISK) must contain risk_level, risk_reason,
    and plain_summary keys with non-empty string values.
  - Kills mutants that rename or remove response dict keys.

TestHighRiskPatterns (10 tests)
  - One test per HIGH_RISK regex: rm -rf, mkfs, dd if=, >/dev/, curl|bash,
    curl|sh, wget|bash, chmod 777.
  - Plus: user_prompt is included in pattern scan (not just normalized_command).

TestWarningPatterns (7 tests)
  - One test per WARNING regex: curl, wget, npm install, pip install,
    git push, git reset --hard.
  - Verifies risk_reason and plain_summary are non-empty strings.

TestSafeFallback (7 tests)
  - Benign commands (ls, cat, grep, git status, echo, empty) return SAFE.

TestNormalizeLevel (6 tests) — unit tests for _normalize_level helper
  - SAFE/WARNING/HIGH_RISK passthrough.
  - HIGHRISK (no underscore) normalization.
  - Lowercase input normalization.
  - Unknown/empty input defaults to WARNING.

TestParseJsonObject (3 tests) — unit tests for _parse_json_object helper
  - Plain JSON, markdown code fence, bare code fence.

=== Remaining 49 survived mutants ===

The 49 remaining are mainly:
  - String literal mutations to error message text (e.g. lowercase/uppercase
    of risk_reason values) — hard to kill without exact string matching, which
    would make tests brittle and tightly coupled to error message wording.
  - A known mutmut v3 runner isolation issue: mutmut copies source to a
    mutants/ dir and runs tests there; async functions with module-level
    imports can be loaded from the original path instead of the mutant, causing
    some kills to not register. This does not affect the value of the test gap
    findings.
